## 1. 文档目标

本文档用于把第二批专题从“follow-up 表格中的一组条目”收成正式范围说明。

本文档重点回答以下问题：

- 第二批到底纳入哪些专题
- 第二批的目标是“冻结什么可信口径”，而不是“把所有实现层细节一次写完”
- 第二批和第一批、第三批、第四批的边界分别是什么
- 第二批完成到什么程度，才可认为经营与成本可信源已经形成正式设计闭环

---

## 2. 第二批范围原则

第二批范围统一采用以下四条原则：

1. 不回头改写第一批已经冻结的主对象与主事实。
2. 只处理会直接影响 `L2/L4/L5` 经营可信度、历史回看稳定性和提成解释一致性的专题。
3. 先固定经营与成本可信口径，再完成桥接映射并写回实现设计总文档。
4. 不把第三批的流程健壮性增强项和第四批的范围限制项混入第二批。

## 5. 第二批入场与出场条件

### 5.1 入场条件

第二批开始前应满足：

1. 第一批六个专题不存在方向级未决策问题。
2. 第一批七层实现映射已补到可稳定支撑第二批继续下钻的程度。
3. `phase2-review-follow-up-plan.md` 与本文件对第二批专题范围保持一致。
4. 第二批主回写文档已明确，不再继续拆散成临时讨论结论。

### 5.2 出场条件

第二批完成后至少应满足：

1. 七个专题均已形成正式规则回写并落到主文档。
2. `phase2-review-follow-up-plan.md` 已同步真实状态。
3. `poms-design-progress.md` 已同步第一批与第二批的当前阶段判断。
4. `phase2-detailed-design-index-map.md` 已提供第二批范围入口。
5. 第二批实现映射桥接入口已固定为 `phase2-second-batch-implementation-mapping.md`，且六份实现设计总文档已完成首轮写回，不再停留在桥接层重复定义第二批专题。

---

## 6. 当前结论

第二批当前已经不适合再以“零散补点”方式推进，而应被视为正式的经营与成本可信源范围，并已进入“总文档首轮写回后继续收敛”的阶段。

在此基础上：

- 第一批负责主事实和主边界的稳定落地
- 第二批负责把经营可信口径、历史回看口径和 `L4 -> L5` 动作绑定写稳
- 第三批、第四批继续作为后续增强或范围限制处理

这样，第二阶段后续若继续进入实现设计层，就不会再建立在摇摆的经营解释和历史口径之上。
