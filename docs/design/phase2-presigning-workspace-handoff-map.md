## 1. 文档目标

本文档用于收口签约前六个工作区之间的承接关系。

它重点回答：

- 六个工作区各自的输入来自哪里
- 六个工作区各自向哪里输出正式结论
- 哪些节点会阻断后续推进
- 不投标项目如何通过直接商务路径保持主链连续

本文档的作用不是重复各工作区内部细节，而是把它们统一成一条不会断链的签约前主线。

---

## 2. 六个工作区

当前签约前主线固定由以下六个工作区组成：

1. `立项与推进`
2. `项目总览`
3. `技术与成本`
4. `招投标 / 商务竞标`
5. `报价与毛利评审`
6. `签约就绪`

其中：

- `项目总览` 是驾驶舱和统一入口
- 其余五个是正式工作区
- `签约就绪` 是进入合同主链前的最终收口工作区

---

## 3. 主承接链

### 3.1 标准路径

标准签约前主链建议固定为：

`立项与推进 -> 技术与成本 -> 招投标 / 商务竞标 -> 报价与毛利评审 -> 签约就绪 -> 合同签约`

### 3.2 直接商务路径

如果项目不需要投标，则允许采用：

`立项与推进 -> 技术与成本 -> 报价与毛利评审 -> 签约就绪 -> 合同签约`

此时：

- `招投标 / 商务竞标` 节点不作为阻断项
- 应标记为“不适用”或“已通过直接商务路径替代”

### 3.3 驾驶舱关系

`项目总览` 不插入主承接链，而是横向承接所有工作区摘要：

- 接收所有阶段摘要
- 展示下一步动作
- 展示阻断项
- 提供工作区跳转入口

---

## 4. 各工作区输入输出

### 4.1 `立项与推进`

**输入**:

- 项目基础机会信息
- 销售的初步判断

**输出**:

- 立项结论
- 条件项 / 否决原因 / 待补材料
- 当前是否可进入 `技术与成本`

**阻断规则**:

- 若立项未通过或待补材料未关闭，则不得进入 `技术与成本`

### 4.2 `技术与成本`

**输入**:

- 立项结论
- 项目背景摘要

**输出**:

- 技术可行性结论
- 范围快照
- 风险与保留意见
- 前期成本清单
- 税务成本 / 税负影响说明

**阻断规则**:

- 若技术判断未形成
- 若范围快照未确认
- 若前期成本清单未形成
- 若税务成本仍缺正式表达
  则不得进入后续竞标或报价判断

### 4.3 `招投标 / 商务竞标`

**输入**:

- 技术与成本输出
- 项目总览摘要
- 立项结论

**输出**:

- 竞标形态与阶段
- 竞标结果
- 当前有效版本
- 当前是否可进入 `报价与毛利评审` 或 `签约就绪`

**阻断规则**:

- 对于需要投标的项目：
  - 若结果未形成，则不得跳过进入后续签约判断
- 对于不需要投标的项目：
  - 节点标记为不适用，不构成阻断

### 4.4 `报价与毛利评审`

**输入**:

- `技术与成本`
- `招投标 / 商务竞标` 或直接商务路径
- 项目总览摘要

**输出**:

- 当前报价结论
- 当前审批结论
- 当前是否具备进入 `签约就绪` 的条件

**阻断规则**:

- 若报价评审未放行
- 若仍有关键财务 / 税务 / 回款条件未确认
  则不得进入 `签约就绪`

### 4.5 `签约就绪`

**输入**:

- `立项与推进`
- `技术与成本`
- `招投标 / 商务竞标` 或直接商务路径
- `报价与毛利评审`
- `项目总览`

**输出**:

- 当前是否可签约
- 当前仍缺失的前置项
- 当前可复用到合同的事实

**阻断规则**:

- 若任一必备前置项未完成，则不得进入正式合同签约

### 4.6 `项目总览`

**输入**:

- 上述五个工作区的摘要和状态

**输出**:

- 当前下一步动作
- 当前阻断项
- 当前责任人
- 当前阶段总体判断

**关键规则**:

- `项目总览` 不替代正式工作区
- 它只做摘要、导航与统一解释

---

## 5. 阻断项统一口径

为避免各工作区各说各话，签约前主线统一使用以下阻断项分类：

1. 立项未通过 / 待补材料
2. 技术结论未形成
3. 范围快照未确认
4. 成本清单未形成
5. 税务成本未明确
6. 竞标结果未形成
7. 报价评审未放行
8. 签约前置项未齐

同一项目在 `项目总览` 中展示时，应归并到这组统一口径，而不是每个工作区各自造一套状态词。

---

## 6. 可复用事实统一口径

为避免进入合同时重新录入，签约前主线应至少沉淀以下可复用事实：

1. 客户与项目基础信息
2. 立项结论
3. 范围快照
4. 风险与保留意见摘要
5. 前期成本与税务成本摘要
6. 竞标结果或直接商务路径结论
7. 报价与毛利评审结论
8. 当前回款条件摘要

这些事实应在 `签约就绪` 中汇总，并作为进入合同主链的正式输入。

---

## 7. 当前统一结论

### 7.1 主线已闭合

到当前为止，签约前六个工作区已经全部形成详细草案，主线结构已经闭合。

### 7.2 仍未开始的是模板层

当前签约前主线的下一步，不再是补新的工作区，而是进入 `L1-T06`：

- 前期成本清单模板
- 税务成本表达模板
- 风险分类模板
- 阻断项分类模板
- 关键结论摘要模板

### 7.3 不投标路径已被正式纳入

当前主线已明确支持“不投标直接进入报价与毛利评审”，不会被强制阻断在竞标节点。
