## 1. 目的与基本原则

### 1.1 目的
为规范公司销售提成管理，建立与项目盈利能力、回款情况和流程合规性强关联的激励机制，引导销售行为符合公司长期利益，特制定本方案。

### 1.2 基本原则
- 利润导向原则：销售提成以项目盈利为前提，亏损项目不记提成。
- 回款导向原则：销售提成与实际回款强绑定，未回款不发放。
- 流程合规原则：未按《公司销售规范流程》执行项目，不享有销售提成。
- 责任对等原则：谁主导项目，谁对项目结果负责。

---

## 2. 适用对象与角色定义

### 2.1 适用对象
本方案适用于参与公司项目获取、推进并纳入公司正式销售流程的以下人员：
- 销售人员；
- 经公司确认参与项目的其它提成角色（如售前、战略关系推动等）。

### 2.2 提成角色定义
- 销售负责人：项目第一责任人，对项目从立项至回款全过程负责。
- 售前/技术支持：对项目技术可行性与交付范围负责。
- 战略关系推动人：经批准后介入项目关键决策层的公司高层或外部关键关系人。
- 其他经公司确认的项目支持角色。

---

## 3. 提成计提口径与提成池

### 3.1 提成计提口径
公司项目销售提成以项目贡献毛利为唯一计提口径。

项目贡献毛利=项目实际回款（不含税）−项目直接成本

### 3.2 项目直接成本包括但不限于
- 项目实施相关人力成本；
- 外包、第三方服务费用；
- 差旅、现场实施费用；
- 云资源、软硬件直接采购费用。

### 3.3 提成池计算
项目提成池按以下公式计算：

项目提成池=项目贡献毛利×提成率

---

## 4. 提成率与毛利分档

### 4.1 提成率分档

| 项目贡献毛利率 | 提成率 |
| -------------: | -----: |
|          ＜20% |     0% |
|        20%—30% |     8% |
|        30%—40% |    10% |
|        40%—50% |    12% |
|           ≥50% |    15% |

### 4.2 说明
- 贡献毛利率低于公司红线的项目，不计销售提成。
- 毛利率分档以最终结算数据为准。

## 5. 提成角色分配与冻结

### 5.1 提成角色分配比例

| 角色           |    权重 |
| -------------- | ------: |
| 销售负责人     | 40%—60% |
| 售前/技术支持  | 15%—25% |
| 战略关系推动人 |  5%—15% |
| 公司管理保留   |  5%—10% |

具体比例在项目移交阶段确认并冻结。

### 5.2 角色与权重冻结原则
项目进入“签约与项目移交”阶段前，必须书面确认并冻结：
- 项目销售负责人；
- 参与提成分配的角色；
- 各角色对应权重。

冻结后原则上不得调整，确需调整的，须经公司管理层批准。

## 6. 提成发放规则

### 6.1 提成发放前提
销售提成的发放须同时满足以下基本前提条件：
- 项目已按《公司销售规范流程》完成全部必要审批及移交流程；
- 项目不存在重大流程违规、经营风险或尚未解决的重大争议；
- 项目已产生实际回款，且回款金额符合本章对应阶段的发放条件。

未满足上述任一条件的，公司有权暂停或延迟销售提成的发放。

### 6.2 提成发放节点与比例

#### 6.2.1 提成启动条件
项目须同时满足以下条件，方可启动销售提成发放：
- 项目已完成合同签署并进入实施阶段；
- 项目累计实际回款达到合同不含税金额的20%（含）以上；
- 项目已完成内部项目移交，且未被认定为重大风险项目；
- 项目不存在重大流程违规、经营风险或尚未解决的重大争议。

未同时满足上述条件的，不得启动销售提成发放。

#### 6.2.2 第一阶段提成发放
在满足提成启动条件，且项目推进正常的前提下：

**标准首付款情况（合同首付款比例≥20%）**  
在满足提成启动条件，且合同约定首付款比例不低于20%的情况下，第一阶段销售提成按以下档位执行：

- 基础档（20%）  
  项目累计实际回款达到不含税金额20%，且项目已正常启动并进入实施阶段，可累计发放至应得销售提成总额的20%；
- 中档（25%）  
  在前述基础上，项目实施计划明确，经技术支持或项目负责人确认具备可执行性的，可累计发放至25%；
- 上限档（30%）  
  在前述基础上，项目回款节奏正常，且未因销售承诺问题被列为重点风险项目的，可累计发放至30%。

**低首付款情况（合同首付款比例＜20%）**
- 第一阶段累计发放比例原则上不超过应得销售提成总额的10%；
- 对客户信用良好、实施风险低、付款条件明确且回款节奏稳定的项目，经公司管理层批准，第一阶段累计发放比例可提高至不超过15%；
- 在未解除低首付款约束前，不得适用标准首付款情况下的第一阶段提成档位。

**低首付款持续约束与解锁规则（50%解锁点）**
- 对于低首付款项目，在项目累计实际回款达到合同不含税金额50%（含）以上前，持续适用低首付款提成发放规则；
- 当低首付款项目累计实际回款达到不含税金额50%（含）以上，且项目回款节奏稳定、项目推进正常的，经公司确认后，可解除低首付款约束；
- 低首付款约束接触后，该项目后续提成发放节点与比例，按照标准情况执行。

#### 6.2.3 第二阶段提成发放
在项目达到合同约定的验收节点或阶段性成果，且项目累计实际回款达到合同不含税金额80%（含）以上时：

- 基础档（70%）  
  在完成阶段性交付、项目推进正常的前提下，可累计发放至应得销售提成总额的70%；
- 中档（75%）  
  在前述基础上，已取得客户阶段验收确认，且不存在因销售承诺引发的重大返工或范围争议的，可累计发放至75%；
- 上限档（80%）  
  在前述基础上，项目回款节奏正常，尾款回款条件明确，且不存在尚未解决的重大交付或商务争议的，可累计发放至80%。

#### 6.2.4 最终提成发放
在项目非质保部分回款达到合同不含税金额100%，或合同约定全部履行完成后，可发放除质保金对应比例外的剩余销售提成。

涉及质保金的项目，其质保金对应的销售提成按本方案关于质保金的相关条款执行。

#### 6.2.5 补充说明
- 本条所述各阶段“累计发放比例”为上限值，不自动等同于应发值。
- 销售提成不得提前预支、跨阶段发放或以任何形式变相发放。
- 项目如在后续阶段暴露重大回款异常、交付风险或经营问题，公司有权暂停或调整后续提成发放。

## 7. 质保金项目的销售提成处理
- 对于合同中约定存在质保金的项目，质保金视为项目回款的一部分，其对应的销售提成不随项目验收或非质保回款完成一并发放。
- 项目非质保部分回款达到合同不含税金额100%后，公司可按本方案发放除质保金对应比例外的全部销售提成。
- 质保金对应的销售提成，应在满足以下条件后予以结算和发放：
  - 合同约定的质保期届满；
  - 客户未就项目质量或服务提出尚未解决的重大异议；
  - 质保金实际到账。
- 质保金对应销售提成的比例，原则上与合同质保金占合同不含税金额的比例一致，除非公司另有书面约定。

---

## 8. 离职情况下的销售提成处理
- 销售人员在职期间已按本方案规定条件发放的销售提成，原则上不予追回。
- 销售人员在离职前，项目已满足相应提成发放条件但尚未完成发放的销售提成，视为既得权益，公司应在完成结算后予以发放。
- 销售人员离职时，项目尚未达到提成发放条件的部分，公司不再承担自动发放义务。
- 对于合同约定存在质保金的项目，如销售人员在质保期内离职，且不再承担项目后续客户沟通、问题协调或相关责任的，其质保金对应的销售提成原则上不再发放。
- 经公司确认，离职人员在质保期内仍实际参与并履行相应职责的，可根据其实际承担责任情况，结算部分或全部质保金对应的销售提成。

---

## 9. 回款异常、退款及坏账情况的处理
- 项目回款发生严重逾期、客户明确拒付、合同解除或存在重大退款风险的，公司有权暂停该项目后续销售提成的发放。
- 对已发放销售提成所对应的项目回款，如后续发生退款、冲减、坏账或无法收回的情形，公司有权按实际损失情况，对已发放提成进行调整或扣回。
- 因销售人员在合同签署、付款条件谈判、风险披露等环节存在明显过失，导致项目发生重大回款异常、退款或坏账的，公司可依据本方案及相关管理制度追究相应责任，并对销售提成进行扣减或取消。
- 回款异常、退款及坏账的认定标准，由公司管理层结合财务、商务及项目实际情况予以确认。

---

## 10. 特别说明与管理保留权
- 本方案所述销售提成的发放与结算，均以项目整体风险可控、回款真实有效为前提条件。
- 对于本方案未明确列明的特殊情形，或因客户、政策、市场环境变化等不可预见因素导致的异常情况，公司管理层有权在不违背本方案基本原则的前提下作出合理调整。
- 公司行使管理保留权的目的在于维护公司整体利益与制度公平性，不作为常规处理方式。
- 本方案中涉及的管理裁量与例外处理，应形成书面记录，作为后续审计与追溯依据。

## 11. 责任、违规与制度执行

### 11.1 责任归属原则
- 销售提成的取得以责任承担为前提。
- 项目销售负责人为项目第一责任人，对项目的合同条件、付款结构、风险披露、客户沟通及最终回款结果承担主要责任。
- 经确认参与提成分配的其他角色，应在其职责范围内对相应工作成果与风险承担责任。
- 未实际参与项目推进、交付协调或客户维护工作的人员，不得主张销售提成权益。

### 11.2 销售违规行为认定
存在下列情形之一的，视为销售违规行为：
- 未按《公司销售规范流程》完成立项、评审、审批即推进项目；
- 绕过公司审批流程，私自对客户做出价格、范围、交付或付款承诺；
- 故意隐瞒项目风险、真实成本或客户付款条件；
- 为促成签约而提供明显不合理的付款结构或价格条件；
- 因销售承诺不当，导致项目返工、纠纷或重大交付风险；
- 在项目回款过程中消极怠工，未履行合理的汇款推进义务。

### 11.3 违规处理与提成影响
- 对于一般违规行为，公司有权对相关项目销售提成采取以下一种或多种处理措施：
  - 暂停提成发放；
  - 降低提成比例；
  - 取消未发放的销售提成。
- 对于造成项目严重亏损、坏账或重大法律纠纷的违规行为，公司有权取消该项目全部销售提成，并依据公司相关制度追究进一步责任。
- 违规行为一经认定，不因人员离职、岗位调整或项目阶段变化而自动免责。

### 11.4 责任减免与例外情况
- 因客户原因、政策变化或不可抗力因素导致项目回款或执行异常的，经公司确认，可不视为销售违规行为。
- 销售人员已合理履行风险提示义务，并留有书面或系统记录的，可作为责任减免的重要依据。
- 责任减免或例外处理应形成书面结论，并纳入项目档案留存。

### 11.5 提成角色冻结与责任绑定
- 项目提出角色与分配比例在项目移交阶段确认并冻结后，与项目责任一并绑定。
- 项目后续阶段新增人员介入，不当然取得销售提成权益，除非经公司书面确认。
- 已冻结的提成角色，如未履行相应责任，公司有权调整或取消其对应提成。

### 11.6 制度执行与监督
- 本方案由公司统一执行，财务部门负责提成核算与发放审核。
- 销售提成的核算、发放及调整结果，应留存完整记录，接受内部审计或管理层复核。
- 任何个人不得以口头承诺、个人协议等方式变相改变本方案的执行结果。

### 11.7 争议处理与解释权
- 因销售提成产生争议的，应首先依据本方案条款进行认定。
- 对条款理解存在分歧的，由公司管理层最终解释。
- 本方案解释结果适用于同类情形，不作为个案临时处理。

### 11.8 制度生效与持续优化
- 本方案自2026年1月1日起试行，与《公司销售规范流程》具有同等约束力。
- 公司可根据经营实际与执行反馈，对本方案进行修订和完善。
- 修订后的方案经发布后生效，原则上不追溯既往。
