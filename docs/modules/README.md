# 模块详细设计

本目录用于存放各模块的详细设计文档。

这些文档应建立在以下文档之上：

- 业务制度源文档
- `docs/design/poms-requirements-spec.md`
- `docs/adr/`
- `docs/design/poms-hld.md`

## 建议优先级

第一批建议优先补充：

- 用户管理
- 角色与权限
- 组织单元
- 导航菜单
- 销售线索登记
- 项目立项评估
- 技术可行性与范围确认
- 报价与毛利评审
- 合同签约与项目移交
- 提成计算与结算

## 建议模板

每篇模块设计文档建议至少包含：

- 背景与目标
- 关联业务制度条款
- 用户角色与权限
- 页面与交互范围
- 数据模型
- 状态流转
- API 草案
- 异常与边界场景
- 测试关注点
