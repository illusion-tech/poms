## 1. 文档目标

本文档用于集中沉淀 `POMS` 第一阶段已经在设计文档中收敛的状态和值域口径，避免同一状态词在不同文档中表达不同含义。

本文档当前只覆盖已经较稳定的状态集，不追求一次性囊括所有领域对象的全部细化枚举。

---

## 2. 设计文档状态

| 状态                          | 含义               | 适用说明                               |
| ----------------------------- | ------------------ | -------------------------------------- |
| `Draft`                       | 草稿中             | 方向已开始成形，但尚未形成稳定基线     |
| `Draft (Baseline)`            | 草稿基线           | 已形成首版稳定基线，但仍需继续收口     |
| `Review`                      | 评审中             | 已进入评审或评审前收口阶段             |
| `Accepted`                    | 已接受             | 已作为当前阶段正式设计依据             |
| `Accepted (Phase 1 Baseline)` | 第一阶段基线已接受 | 表示在第一阶段范围内已被接受的正式依据 |
| `Active`                      | 持续维护中         | 常用于进度板、清单、评审摘要等治理文档 |
| `Archived`                    | 已归档             | 历史保留，不再作为当前设计输入         |

## 3. `Project` 生命周期字典

### 3.1 `Project.stage`

| 值                   | 含义                 |
| -------------------- | -------------------- |
| `assessment`         | 立项评估             |
| `scope-confirmation` | 技术可行性与范围确认 |
| `commercial-closure` | 商务收口             |
| `contracting`        | 签约                 |
| `handover`           | 项目移交             |
| `execution`          | 正式执行             |
| `acceptance`         | 验收确认             |
| `completed`          | 完成                 |
| `closed-lost`        | 关闭丢单             |
| `closed-terminated`  | 关闭终止             |

### 3.2 `Project.status`

| 值                 | 含义                                         |
| ------------------ | -------------------------------------------- |
| `active`           | 当前阶段正常推进中                           |
| `pending-approval` | 当前阶段存在待审批动作，未完成前不得放行     |
| `blocked`          | 因缺资料、审批未通过、规则不满足等原因被阻断 |
| `on-hold`          | 因例外原因被人工挂起                         |
| `completed`        | 当前阶段已完成                               |
| `closed`           | 项目已关闭，不再参与正常推进                 |

说明：

- `stage` 表达主链路位置
- `status` 表达该位置下的推进状态
- `BidProcess.status` 不应直接塞入 `Project.status`

## 4. 平台治理状态字典

### 4.1 用户账户状态

| 对象   | 状态值     | 含义                                     |
| ------ | ---------- | ---------------------------------------- |
| `User` | `active`   | 可参与登录、资料读取、角色计算、导航生成 |
| `User` | `inactive` | 不得继续登录；下一次请求必须被后端拒绝   |

### 4.2 权限与角色生命周期

| 对象         | 状态值       | 含义                                                       |
| ------------ | ------------ | ---------------------------------------------------------- |
| `Permission` | `active`     | 可被角色正常分配                                           |
| `Permission` | `inactive`   | 不再允许新增分配，但第一阶段默认保留存量有效关系的授权效力 |
| `Permission` | `deprecated` | 已废弃，不允许再被新角色或新变更使用                       |
| `Role`       | `active`     | 参与正常授权计算                                           |
| `Role`       | `inactive`   | 保留历史记录，但不再参与新增分配和后续授权计算             |

### 4.3 关系实体状态

| 对象                       | 状态值    | 含义                       |
| -------------------------- | --------- | -------------------------- |
| `RolePermissionAssignment` | `active`  | 当前有效的角色权限关系     |
| `RolePermissionAssignment` | `revoked` | 已撤销关系，但保留历史记录 |
| `UserRoleAssignment`       | `active`  | 当前有效的用户角色关系     |
| `UserRoleAssignment`       | `revoked` | 已撤销关系，但保留历史记录 |
| `UserOrgMembership`        | `active`  | 当前有效的用户组织关系     |
| `UserOrgMembership`        | `revoked` | 已撤销关系，但保留历史记录 |

### 4.4 导航-路由映射状态

| 值            | 含义                                                             |
| ------------- | ---------------------------------------------------------------- |
| `implemented` | 目标导航链接已有真实页面承载，可按正式链接启用                   |
| `bridged`     | 目标链接与当前前端现状暂不完全一致，但已有兼容跳转或迁移桥接方案 |
| `planned`     | 目标链接已确定，但页面尚未就位，不应正式启用                     |

说明：

- 从 `planned` 进入 `bridged` 或 `implemented` 前，应补充页面承载与守卫验证
- `platform.users` 在真实页面承载未就位前维持 `planned`，不提前记为 `bridged`

## 5. 提成治理状态字典

### 5.1 `CommissionRoleAssignment`

| 值     | 含义                     |
| ------ | ------------------------ |
| 草稿   | 允许录入与编辑           |
| 待冻结 | 已提交冻结但尚未完成放行 |
| 已冻结 | 形成正式冻结版本         |
| 变更中 | 已发起变更流程           |
| 已替代 | 已被新版本替代           |
| 已作废 | 当前版本已作废           |

### 5.2 `CommissionCalculation`

| 值         | 含义             |
| ---------- | ---------------- |
| 待计算     | 尚未形成计算结果 |
| 已计算     | 已完成初算       |
| 待复核     | 等待复核放行     |
| 已生效     | 结果已生效       |
| 已冻结     | 当前结果已冻结   |
| 已重算替代 | 已被新结果替代   |
| 已作废     | 当前结果已作废   |

### 5.3 `CommissionPayout`

| 值     | 含义               |
| ------ | ------------------ |
| 草稿   | 已创建发放草稿     |
| 待审批 | 等待审批放行       |
| 已批准 | 已完成发放审批     |
| 已发放 | 已登记业务发放     |
| 已暂停 | 因异常被暂停       |
| 已冲销 | 已通过冲销动作处理 |
| 已关闭 | 当前发放记录关闭   |

### 5.4 `CommissionAdjustment`

| 值     | 含义           |
| ------ | -------------- |
| 草稿   | 已创建调整草稿 |
| 待审批 | 等待审批放行   |
| 已批准 | 已审批通过     |
| 已执行 | 已完成执行     |
| 已驳回 | 已被驳回       |
| 已关闭 | 当前调整单关闭 |

### 5.5 `CommissionRuleVersion`

| 值     | 含义               |
| ------ | ------------------ |
| 草稿   | 规则草稿中         |
| 待生效 | 已提交生效         |
| 已生效 | 当前规则版本已生效 |
| 已停用 | 当前规则版本已停用 |

## 6. 使用约定

- 同一状态词在不同对象上可以重复出现，但含义应保持相邻一致，不得出现相反解释
- 若某对象需要新增状态，应优先在该对象详细设计中收敛，再决定是否提升到本字典
- 若某状态只在单一模块临时出现，且不具备跨文档复用价值，则不应急于加入本字典
