# POMS ADR

本目录用于记录 `POMS` 的架构决策记录（Architecture Decision Record, ADR）。

这些文档用于在功能设计和实现之前，先固定高影响、难回退的设计决策，避免后续在接口、数据模型、权限判断和前端信息架构上反复返工。

## 当前 ADR 列表

- `ADR-001`: 平台权限模型与授权边界
- `ADR-002`: 组织单元建模与授权关系
- `ADR-003`: 导航菜单单一可信源策略
- `ADR-004`: 合同资金域后端模块边界
- `ADR-005`: 一期审批流实现策略
- `ADR-006`: 第一阶段主对象正式命名为 Project
- `ADR-007`: 第一期财务联动与业务登记边界
- `ADR-008`: 当前用户资料输出契约
- `ADR-009`: 平台导航父组可见性规则
- `ADR-010`: 平台用户管理路由桥接状态
- `ADR-011`: 招投标与 Project 生命周期的建模关系

## 约定

- 文件命名使用 `NNN-短标题.md`
- 状态建议使用：`提议中 (Proposed)`、`已接受 (Accepted)`、`已废弃 (Deprecated)`
- ADR 关注“为什么这样定”，不替代详细功能设计文档
- 若决策已变更，应新增后续 ADR，而不是静默改写旧结论
